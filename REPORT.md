## Problem Statement
Data — Retrieve up-to-date information for the top ten stock markets in the world according to https://www.fool.com/research/largest-companies-by-market-cap/
PDF —  Generate pdfs with the strucual data

Delivery — The PDF will be delivered to a WhatsApp chat automatically as a document attachment


## Methodology

Data Source
The top 10 is defined by top 10 companies with largest market capitalization in May 2026 according to
https://www.fool.com/research/largest-companies-by-market-cap/

Stock Tickers used: NVDA, GOOG, AAPL, MSFT, AMZN, TSM, AVGO, META, TSLA, WMT

Data are collected through yfinance libary in python 
For each company, the following fields are retrieved:
- Ticker, Company Name, Current Price, Daily Price Change (%)
- Day Low / Day High, 52-Week Low / 52-Week High
- Volume, Average Volume, Bid, Ask
- Beta, P/E Ratio, Market Capitalisation (in USD trillions), Currency

After collecting the data, the data are appended as dictionaries to a list, and then converting to a panda data frame


The PDF is then generated by report lab
The following are contained:
- Author name
- Title
- Time of generation
- 2 page of tables of the stock data

The automation:
1. Opens a persistent browser session (login saved after first QR scan unless user log out for the specific broswer)
2. Searches for the recipient (only possible when the user is part of the contact of the account)
4. Attaches the generated PDF through the document attachment 
5. Sends the message and closes the browser 

## Experimental Results
- All 10 tickers successfully return live market data via yfinance
- PDF is generated correctly with title, timestamp, author name, and 2 page tables
- Whatsapp delivery is received in testing

## Limitations
- WhatsApp Web selectors may break if the code are not set properly for the languages, since the name for the buttons are different for different languages users
- Requires login and logout by users instead of the system code
- yfinance is an unofficial library and may be inaccurate
- deliver is only possible when the recipient is in the contact of the user
- The set up can only works to person but not group chats
